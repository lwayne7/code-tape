# 26飞书工程训练营-前端组-实现一个代码讲解工具

## 一、项目简介

基于 Web 技术的代码讲解录制与回放平台，支持录制者在内置代码编辑器中编写代码的同时，同步采集语音与摄像头，完整记录键盘输入、鼠标轨迹、文本选区等操作行为，最终生成一个可交互回放的讲解内容。回放时，观看者可以像"看直播回放"一样还原讲解者的完整操作过程，支持倍速播放、进度跳转、快捷键展示、字幕等功能。

适用场景：算法讲解、代码 Review、技术分享、面试复盘、编程教学等。

## 二、项目目标

- **P0**：一个月内完成可用 Demo，支持录制、回放核心流程，支持展示效果
- **P1**：完善交互体验与云端存储逻辑，提升性能与细节体验（Nice to Have）
- **加分项**：WebRTC 实时面试模式、本地 AI 字幕生成与纠错

## 三、技术要点说明

本项目有几个核心技术挑战，需要在项目初期充分调研并做好方案选型

- **代码编辑器的选型与集成**：需要一个可嵌入网页的代码编辑器组件，要求支持语法高亮、多语言，并能够监听每一次内容变更事件。学生需要自行调研市面上的开源方案，理解各方案的优缺点，并完成集成。
- **操作序列的录制与回放引擎**：这是项目最核心的架构挑战。录制过程产生的不是视频，而是一系列带时间戳的事件流。回放引擎需要按时间轴调度这些事件，并正确处理倍速、快速定位、暂停等交互。

### 沙箱执行，以下二选一

- **后端代码沙箱执行**：后端负责接收并执行用户的代码，需要理解直接执行与隔离执行之间的安全差异，并至少实现基本的资源限制。
- **前端代码页面展示**：前端直接展示页面的效果，类似 https://jsbin.com/ 或 https://codesandbox.io/ 效果

## 四、项目任务拆解

### 【P0】主流程功能

#### 1. 代码编辑器模块

- 支持 JavaScript / TypeScript 语法高亮与基础补全
- 支持常见编辑器快捷键（格式化、注释、跳转等）
- 提供简洁的编辑器工具栏（语言切换、字号等基础设置）
- 支持提交代码至后端执行，展示运行结果（stdout / stderr）或支持在前端展示运行结果

#### 2. 操作录制模块

需要录制的事件类型包括：

- **内容变更事件**：每次编辑器内容发生变化时，记录变更后的完整代码内容
- **光标与选区事件**：记录光标位置与选中范围
- **鼠标轨迹事件**：记录鼠标在编辑器区域的坐标变化（可降低采样频率以减少数据量）
- **快捷键事件**：记录用户触发的组合键（如 Ctrl+Z、Ctrl+/ 等），区别于普通字符输入单独捕获

> 注意：普通字符输入已经通过内容变更事件覆盖，快捷键事件主要捕获不产生字符的控制类操作。

#### 3. 音视频录制模块

- 采集麦克风音频（MediaStream）
- 支持开关麦克风
- 支持采集摄像头视频流，提供悬浮预览窗口，支持拖拽移动位置
- 支持开关摄像头
- 支持切换录制设备

#### 4. 总控模块

- 支持开始 / 暂停 / 继续 / 结束录制
- 录制过程中实时显示录制时长
- 结束录制后，将操作事件序列与音视频数据打包，提示用户保存或上传

#### 5. 回放模块

根据事件序列还原编辑器状态，支持：

- 倍速播放（0.5x / 1x / 1.5x / 2x）
- 暂停 / 继续
- 拖动进度条跳转到任意位置（seek）
- 音量调节与静音

回放时同步展示：

- 编辑器内容的变化过程
- 鼠标位置（以"激光笔"效果展示）
- 光标与选区
- 快捷键浮层：在画面右下角以悬浮 Badge 展示当前触发的快捷键，短暂停留后淡出
- 摄像头视频（若录制时有开启）

### 【P1】增强体验与扩展功能

#### 1. UI 与交互优化

- 支持深色 / 浅色主题切换
- 适配 PC 端布局，操作区与回放区清晰分区
- 回放时，快捷键展示支持用户手动开关

#### 2. 云端回放中心

- 录制完成后可上传至后端，保存录制记录
- 支持列表查看所有录制，展示标题、时长、录制时间
- 支持在线播放 / 删除 / 重命名

#### 3. 后端代码执行（安全增强）

- 基础实现：后端接收 JS/TS 代码，执行后返回 stdout / stderr
- 进阶实现：增加执行超时限制、内存限制，防止死循环或资源耗尽
- 挑战项⭐⭐：理解并实现进程级或容器级隔离，避免用户代码影响宿主系统（可调研 V8 Isolate、QuickJS WASM、Docker 容器等方案）

#### 3. 前端代码执行（安全增强）

- 基础实现：前端接收 JS/TS/CSS/HTML 代码，执行后展示结果，注意不要污染界面的原本逻辑
- 进阶实现：运行环境拦截 Console 输出，支持在展示结果的同时展示 console 输出内容，支持
- 挑战项⭐⭐：理解并实现前端的沙箱环境，增加安全限制，如禁止弹窗等

### 【P1+】进阶功能（加分项）

#### 1. WebRTC 实时面试模式 ⭐⭐

模拟远程面试场景，面试官与被面试者同时在线：

- 被面试者端：正常录制，代码编辑器中的所有操作实时同步至面试官端
- 面试官端：实时查看被面试者的编辑器状态（操作序列实时推送），同时通过 WebRTC 进行双向音视频通话
- 面试结束后，被面试者端可将本次录制保存为回放
- 挑战项⭐⭐⭐：处理网络抖动下的事件序列时序同步，保证面试官侧看到的状态与被面试者侧一致

#### 2. 本地 AI 字幕生成 ⭐⭐

- 录制结束或回放前，将音频送入语音识别模型，生成带时间戳的字幕文件
- 字幕在回放时展示于编辑器下方，随播放进度滚动
- 支持点击字幕跳转至对应时间点
- 模型推荐在本地或后端运行（可调研 Whisper.cpp 或其 WASM 版本）
- 挑战项⭐⭐：接入本地 LLM 对识别结果进行纠错（如专业术语、变量名的修正）
- 挑战项⭐⭐⭐：基于字幕内容自动分段，生成章节跳转点（如"第一段：问题分析"，"第二段：代码实现"）
